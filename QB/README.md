# QB (Quarterback) Operating Layer

Project: __PROJECT__

## Canon Operating Model
- Coach: `UXOS/00` (portfolio orchestration and governance).
- Quarterback (QB): this repo's `00_QB` thread (`00` folder) for local planning and acceptance.
- `QB/`: required interface between Coach and project QB for status, dispatch, and reporting.

## Purpose
QB is the local execution layer for planning, dispatch, and reporting.
It keeps this repo aligned with UXOS ways of working.

## Commands
- `./QB/qb check` : governance and repo-health checks
- `./QB/qb plan` : generate next-action dispatch pack
- `./QB/qb report` : generate daily heartbeat summary
- `./QB/qb sync` : read recent git changes into a status-review note
- `./QB/qb instrument` : manage web instrumentation pack (ui capture + marketing + research)
- `./QB/qb ffa-state-migrate` : backfill `epistemic_state` on framework records (`seed` default)
- `./QB/qb ffa-plan-sync` : sync FFA changes to plan alignment state and audit queue bindings
- `./QB/qb ffa-reconcile` : run full FFA reconciliation after canonical changes and emit drift/gap report
- `./QB/qb ack-update --by <name>` : acknowledge pending Coach update gate before work
- `./QB/qb supabase-env show|set-ref|sync` : reference shared Supabase env and sync into local `.env.local`
- `./QB/qb compact` : package FFA-only compaction prompt from conversation context
- `QB/THREAD_INTRO_STANDARD_00_08.md` : required first-message standard for bootstrap threads 00-08

## QB-to-QB Rule
- QB-to-QB communication is allowed only via QB dispatch artifacts.
- Coach retains accountability for cross-repo sequencing.
- Cross-repo scope/contract/architecture changes require Coach escalation.

## Control Loop (Required)
1. Chief dispatches using canonical handoff format.
2. PM executes and updates `QB/status.json` and `QB/WORK_QUEUE.md`.
3. PM emits `QB/report_YYYYMMDD.md`.
4. Chief reviews outputs and reprioritizes.

## QB Update Model (Hybrid)
1. Event-based updates (required)
- Update `QB/status.json` on goal changes, blockers, major decisions, scope shifts, milestone changes.

2. Daily heartbeat (required)
- Generate one daily report from `QB/status.json`.

3. Prompt-driven deep updates (as needed)
- Use explicit planning prompts for pivots/replans.

## Autonomy and Queueing
- Use `QB/AUTONOMY_MATRIX.md` to decide if work can run autonomously.
- If not autonomous, add item to `QB/WORK_QUEUE.md` before execution.

## Source of Truth
- `QB/status.json` is canonical.
- `QB/DECISION_LOG.md` is canonical for decisions.
- Git changelog is advisory context only.

## Output Conventions
Every dispatch generated by QB must follow:
1. Project
2. Destination
3. Prompt
4. Acceptance
5. Return

## Primary Model + Matrix
- Portfolio primary model is owned by Coach in UXOS.
- This project must maintain `QB/ARTIFACT_THREAD_MAP.json` as a scoped subset.
- Every active artifact type must map to a valid primary thread.

## Primary Model + Artifact/Thread Matrix (Required)
- `QB/status.json` must include `project_type`.
- `QB/ARTIFACT_THREAD_MAP.json` is mandatory and maps active artifact types to a primary thread.
- Every active artifact type must map to one valid `primary_thread` (`00_QB`, `01`..`09`).
- `00_QB` provides control visibility across all artifact classes.
- Portfolio-level full matrix is maintained in UXOS; project repos maintain scoped subsets only.

## FFA Epistemic State (Required)
- Every framework foundation record must include `epistemic_state`.
- Allowed values: `seed`, `hypothesis`, `stabilized`.
- Use `./QB/qb ffa-state-migrate` before `./QB/qb check` when onboarding older FFAs.
- OST is first-class in FFA and must include arrays:
1. `desired_outcomes`
2. `opportunities`
3. `solutions`
4. `ost_links`
- Seed records must include `seed_metadata`:
1. `seed_label`
2. `evaluation_status` (`research_backed|research_needed`)
3. `probe_intent`

## FFA <-> Plan Alignment (Required)
- Any FFA change requires `./QB/qb ffa-plan-sync` before check/report.
- Every active `QB/WORK_QUEUE.md` row must include either:
1. `ffa_ref:<record_or_bundle_id>`
2. `no_ffa_impact_reason:<short_reason>`
- `./QB/qb check` blocks if FFA hash is unsynced or active queue rows are missing alignment markers.
- After major FFA changes, run `./QB/qb ffa-reconcile --scope full` before new feature execution.
- Reconciliation policy: `QB/policies/ffa_reconciliation_policy.md`

## Web Instrumentation Pack
- Included by default in bootstrap; activation is explicit via `./QB/qb instrument`.
- Tracks:
1. UI capture (`QB/WEB_INSTRUMENTATION_STATUS.json`, `QB/WEB_CAPTURE_TARGETS.json`)
2. Marketing telemetry (`QB/MARKETING_TELEMETRY_STATUS.json`, `QB/MARKETING_TELEMETRY_SCOREBOARD.md`)
3. Research loop (`QB/RESEARCH_INSTRUMENTATION_STATUS.json`, `QB/RESEARCH_EXPERIMENT_REGISTER.md`, `QB/RESEARCH_LEARNING_SCOREBOARD.md`)

## Steady-State Handoff Rule (Required)
- QB does the work directly when it can.
- QB gives copy/paste prompt handoffs only when work must happen in another thread/tool.
- Every handoff must use exact format: `Project / Destination / Prompt / Acceptance / Return`.
